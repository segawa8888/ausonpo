開発の流れ、フォルダ構成等

開発の流れ

  • 初回はgitからファイルをpullし「npm install」を実行して下さい。
  • 「npx gulp」でタスクランナーが起動します。
  • localhostが起動します。(http://localhost:8080/)
  • HTMLファイルのフォルダパス、ファイル名はディレクトリリスト参照。
  • HTMLテンプレートについてはこちらを参照。
  • ファーストビュー内の画像には「loading="eager"」以降の画像は「loading="lazy"」を指定する。
  • pictureタグを使用してWebP対応を行う。 
    
<picture>
  <source srcset="/assets/img/common/header_logo.webp" type="image/webp" />
  <img src="/assets/img/common/header_logo.png" alt="au損保" loading="eager" />
</picture>
  • gulpを止めるとエラーが出ますが開発上は問題はありません(後日修正します・・・)。

画像の書き出しについて

  • 画像ファイルは/src/img/フォルダに保存して下さい。同名の圧縮した画像ファイルおよびWebP形式のファイルが/assets/img/フォルダに出力されます。
  • 共通で使用する素材はcommonフォルダに、各ページ固有のファイルはページのカテゴリ名のフォルダを作成して保存。

Sassのコーディングについて

  • ソースは/src/sass/。保存すると/assets/フォルダにコンパイルされます。
  • FLOCSSでコーディング。
  • HTMLコーディングの負担軽減のため、modifierは「.-モディファイア名」の形式を採用しています。
  • 基本的に4の倍数でコーディング。数値から外れている要素は近似値で対応。
  • コンテナも現状は1100px、888px、654pxの3サイズ。数値から外れている要素は近似値で対応。
  • レスポンシブ対応のため、px単位のプロパティは全て「rem()」関数を使用して下さい。(px→remに変換されます) 
    
.c-heading1-bg-primary {
  font-size: rem(26px);
  padding: rem(14px) 1em;
  margin-bottom: rem(32px);
}
  • 背景画像は、WebP非対応ブラウザの場合bodyに.no-webp-supportクラスが付与されるので、以下のように対応する。 
    
background-image: url(../img/common/header_icon_login_pc.webp);
.no-webp-support & {
  background-image: url(../img/common/header_icon_login_pc.png);
}

JavaScriptのコーディングについて

  • ソースは/src/js/script/。保存すると/assets/フォルダにコンパイルされます。
  • スライダー系を追加する場合は/src/js/script/05_slider.jsにまとめて下さい。
  • 新しい機能を実装する際は「連番_機能名.js」でファイルを追加して下さい。
